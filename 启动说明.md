# backend 与 ai-service 启动说明

本文说明当前项目中 `ai-service` 与 `backend` 的本地启动方式，以及它们之间的联调关系。

## 1. 项目结构

- `backend`：Spring Boot 后端，默认端口 `8080`
- `ai-service`：FastAPI AI 服务，默认端口 `8001`

调用关系如下：

1. 前端或接口测试请求 `backend`
2. `backend` 在触发日记 AI 分析时，请求 `ai-service`
3. `ai-service` 再调用 DeepSeek 接口生成分析内容

## 2. 启动前准备

### 2.1 backend 环境

请先准备以下环境：

- JDK 17
- Maven Wrapper 已内置，无需额外安装 Maven
- MySQL 8.x

当前 `backend` 默认数据库配置在 `backend/src/main/resources/application.properties`：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/psychology_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
spring.datasource.username=root
spring.datasource.password=xyk200406
```

建议先确认本地已存在数据库：

```sql
CREATE DATABASE psychology_db DEFAULT CHARACTER SET utf8mb4;
```

说明：

- `spring.jpa.hibernate.ddl-auto=update` 已开启，表结构会在启动时自动更新
- 如果你不想使用当前账号密码，请先修改 `application.properties`

### 2.2 ai-service 环境

请准备：

- Python 3.12 或兼容版本

安装依赖：

```powershell
cd ai-service
py -m pip install -r requirements.txt
```

## 3. 启动 ai-service

### 3.1 配置环境变量

先在 `ai-service` 目录下复制一份环境变量文件：

```powershell
cd ai-service
copy .env.example .env
```

然后编辑 `ai-service/.env`，至少填写：

```env
DEEPSEEK_API_KEY=你的DeepSeekKey
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_TIMEOUT_SECONDS=30
AI_SERVICE_INTERNAL_TOKEN=
AI_SERVICE_MAX_OUTPUT_TOKENS=320
```

说明：

- `DEEPSEEK_API_KEY` 必填，否则 AI 分析接口会返回 `503`
- `AI_SERVICE_INTERNAL_TOKEN` 可选；如果填写了，`backend` 必须配置同一个值

### 3.2 启动命令

在 `ai-service` 目录执行：

```powershell
py -m uvicorn main:app --host 0.0.0.0 --port 8001 --reload
```

启动成功后可访问：

- 健康检查：[http://localhost:8001/health](http://localhost:8001/health)

如果返回类似结果，说明 `ai-service` 已正常启动：

```json
{
  "status": "ok",
  "model": "deepseek-chat"
}
```

## 4. 启动 backend

`backend` 通过 `application.properties` 读取 `ai-service` 地址：

```properties
ai-service.base-url=${AI_SERVICE_BASE_URL:http://localhost:8001}
ai-service.analyze-path=${AI_SERVICE_ANALYZE_PATH:/internal/v1/mood/analyze}
ai-service.internal-token=${AI_SERVICE_INTERNAL_TOKEN:}
ai-service.connect-timeout-ms=${AI_SERVICE_CONNECT_TIMEOUT_MS:3000}
ai-service.read-timeout-ms=${AI_SERVICE_READ_TIMEOUT_MS:30000}
```

如果你在 `ai-service/.env` 中配置了 `AI_SERVICE_INTERNAL_TOKEN`，启动 `backend` 前也需要在当前终端设置同一个值，例如：

```powershell
$env:AI_SERVICE_INTERNAL_TOKEN="your_internal_token"
```

如果 `ai-service` 不在 `8001` 端口，也需要同步设置：

```powershell
$env:AI_SERVICE_BASE_URL="http://localhost:8001"
```

然后在 `backend` 目录执行：

```powershell
cd backend
.\mvnw spring-boot:run
```

启动成功后默认访问地址：

- 后端服务：[http://localhost:8080](http://localhost:8080)

## 5. 联调顺序

建议按下面顺序启动：

1. 先启动 MySQL
2. 再启动 `ai-service`
3. 最后启动 `backend`

原因：

- `backend` 调用 AI 分析接口时依赖 `ai-service`
- 如果先启动 `backend`，服务本身能起来，但调用日记 AI 分析时会报 `AI 服务暂不可用`

## 6. 联调验证

### 6.1 验证 ai-service

先验证健康检查：

```powershell
curl http://localhost:8001/health
```

### 6.2 验证 backend

确认 `backend` 启动后，可以继续走现有业务流程：

1. 注册或登录
2. 创建一篇日记
3. 调用 `POST /api/diary/{id}/ai-analysis`

说明：

- 该接口仍然保持不变
- 成功后，分析结果会写回数据库中的 `ai_analysis` 字段
- 如果 DeepSeek 调用失败，接口会报错，且不会覆盖原有分析内容

## 8. 常见问题

### 8.1 `ai-service` 提示 `DeepSeek API Key 未配置`

原因：`ai-service/.env` 中没有填写 `DEEPSEEK_API_KEY`。

解决：补上正确的 DeepSeek Key 后重新启动 `ai-service`。

### 8.2 `backend` 提示 `AI 服务连接超时，请稍后重试`

常见原因：

- `ai-service` 没启动
- `AI_SERVICE_BASE_URL` 配置错误
- `ai-service` 端口不是 `8001`

### 8.3 `401 未授权访问 AI 服务`

原因：`ai-service` 设置了 `AI_SERVICE_INTERNAL_TOKEN`，但 `backend` 没有配置相同的 `AI_SERVICE_INTERNAL_TOKEN`。

### 8.4 backend 无法连接数据库

请检查：

- MySQL 是否启动
- `psychology_db` 是否已创建
- `backend/src/main/resources/application.properties` 中的用户名密码是否正确

## 9. 推荐的本地开发方式

推荐开两个终端：

### 终端 1

```powershell
cd ai-service
py -m uvicorn main:app --host 0.0.0.0 --port 8001 --reload
```

### 终端 2

```powershell
cd backend
.\mvnw spring-boot:run
```

这样修改 Python 或 Java 代码后，都更方便分别观察日志和排查问题。
